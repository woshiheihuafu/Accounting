# 移动设计稿通用规则

## 适用场景

创建、检查、更新、导出或对齐移动端设计稿、`.pen` 文件、截图和设计图片时，使用本规则。

## PRD 与参考图

- PRD 是需求事实来源。
- 竞品截图、参考图和 OCR 结果只能作为布局、信息层级、视觉密度和交互参考。
- 不得把截图中的登录提示、账号提示、营销提示、引流入口或 PRD 未声明功能写入当前项目设计稿。
- 当截图内容与 PRD 冲突时，记录为产品/设计问题并请求确认，不要按截图直接落稿。

## 画布与安全区域

- 移动端设计稿优先使用逻辑尺寸画布，不直接使用物理像素画布；典型 iPhone 画布可使用 `393 x 852 pt` 作为基准。
- 对 @3x 物理像素素材，按逻辑尺寸理解布局；不要把物理 px 直接当作端侧 dp 或 pt。
- 顶部状态栏、动态岛/刘海区域和底部 Home Indicator 区域只适合承载全屏背景或沉浸式媒体，不放置核心内容和功能区。
- 顶部导航、底部导航、工具栏、弹窗、输入框和主要操作必须避开安全区域。

## 布局与触控

- 左右边距建议至少 `16-20pt`，并保持左右对称。
- 元素间距优先使用 `8pt` 倍数，如 `8`、`16`、`24`、`32`。
- 按钮和交互元素最小点击区域为 `44 x 44 pt`。
- 使用 Auto Layout 或等价约束表达固定边距和中间内容拉伸，便于适配 `375pt`、`393pt`、`428pt` 等宽度。
- 避免文字挤压、异常换行、竖排、遮挡和触控区域过小。

## 字体、图标与组件

- 使用目标平台系统字体；iOS 设计可使用 SF Pro 作为基准。
- 常见字号层级可按导航大标题、内容标题、正文/按钮、辅助文字区分；具体字号应服从项目设计系统。
- 图标建议使用 `24 x 24 pt` 容器，并按统一网格绘制内部图形。
- 图标不得大面积使用 `text` 节点冒充；OCR 误识别出的图标文字必须替换为图标节点、path、icon font 或语义相近组件。
- 可复用 UI 应使用 reusable component；发现重复卡片、按钮、标签、导航栏、弹窗等模式时，应抽成可复用组件并统一命名。

## Pencil 节点检查

- 使用 Pencil MCP 检查 `.pen` 的节点、截图、布局快照、导出、变量和设计结构。
- 可点击区域、列表项、顶部栏、底部栏、弹窗、输入区域和状态反馈应有清晰节点层级。
- 导出前检查图标来源、文本节点用途、可复用组件、PRD 覆盖和竞品内容清洗结果。

## 交付检查

- 每个设计页面都应能映射到 `REQ-*`、`SCREEN-*` 或 `FLOW-*`。
- 空态、加载态、错误态和成功态应覆盖关键流程；没有设计覆盖时记录缺口。
- 如果存在 PRD 未声明内容、文字冒充图标、安全区域遮挡或关键状态缺失，不得作为已完成设计交付。
