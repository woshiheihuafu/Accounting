# Coding记账 PRD

## 1. 产品目标

Coding记账参考鲨鱼记账的核心记账体验，先实现一个可用的移动端版本：用户可以在首页查看本月收支摘要和明细流水，可以新增支出或收入账单，可以通过年月/月日选择器切换统计月份或账单日期。

本期重点是“快速记一笔”和“首页明细查看”，暂不实现登录、云同步、报表、预算、资产、发现内容、我的设置等完整能力。

## 2. 功能范围

### 2.1 本期实现

- 首页-明细
  - 展示当前年月、收入合计、支出合计。
  - 展示快捷入口：账单、预算、资产管家、购物返现、更多。
  - 展示登录提示条。
  - 展示按日期分组的账单明细。
  - 展示底部 Tab：明细、图表、记账、发现、我的。
  - 点击中央“记账”入口进入记账页。
- 记账-支出
  - 支持支出/收入 Tab 切换。
  - 本期只实现支出一级类目：餐饮、购物、交通。
  - 选择支出类目后进入金额录入状态。
- 记账-收入
  - 实现收入类目：工资、兼职、理财、礼金、其他、设置。
  - 选择收入类目后进入金额录入状态。
- 账单录入
  - 支持输入金额。
  - 支持填写备注。
  - 支持选择账单日期。
  - 支持完成保存。
- 日期选择
  - 首页左上角月份选择：支持年、月滚动选择。
  - 记账页键盘日期选择：支持年、月、日滚动选择。

### 2.2 本期仅保留入口

以下入口必须保留，但不展开真实功能：

- 首页快捷入口：账单、预算、资产管家、购物返现、更多。
- 底部 Tab：图表、发现、我的。

点击上述入口时，展示“暂未实现”提示或进入空占位页。当前版本不实现预算、资产、返现、报表、发现、我的设置等业务流程。

## 3. 页面与交互

### 3.1 首页-明细

页面结构从上到下：

- 状态栏。
- 黄色品牌头部。
  - 左侧圆形头像/表情入口。
  - 中间标题：Coding记账。
  - 月份选择区：`2026年`、`06月`、下拉箭头。
  - 收入合计：`0.00`。
  - 支出合计：`23.00`。
- 快捷入口卡片。
  - 账单、预算、资产管家、购物返现、更多。
  - 本期均为占位入口。
- 登录提示条。
  - 文案：`请及时登录，避免卸载软件导致数据丢失`。
  - 点击可展示“暂未实现”提示。
- 明细列表。
  - 日期分组标题：`06月16日 星期二`。
  - 分组右侧展示当日支出合计：`支出：23`。
  - 账单项展示类目图标、名称、金额。
  - 示例账单：`牛肉面`，金额 `-23`。
- 底部 Tab。
  - 明细：当前选中。
  - 图表：占位入口。
  - 记账：中央黄色圆形按钮，点击进入记账页。
  - 发现：占位入口。
  - 我的：占位入口。

交互规则：

- 点击月份区域，弹出月份选择器。
- 点击记账按钮，进入记账页，默认展示支出 Tab。
- 点击未实现入口，展示“暂未实现”。
- 首页刷新时按当前选择月份汇总收入、支出和明细。

### 3.2 首页-选择月份弹窗

触发入口：首页左上角年月区域。

弹窗规则：

- 页面背景加深遮罩。
- 底部弹出白色选择面板。
- 顶部操作栏：
  - 左侧：取消。
  - 中间：选择月份。
  - 右侧：确定。
- 选择器包含两列：
  - 年。
  - 月。
- 年、月均可纵向滚动。
- 默认值：`2026 / 6`。

操作规则：

- 点击取消：关闭弹窗，不改变当前月份。
- 点击确定：关闭弹窗，并刷新首页为选中月份。
- 滚动选择时，高亮中间选中行。

### 3.3 记账-支出类目

页面结构：

- 状态栏。
- 黄色顶部栏。
  - 中间 Tab：支出、收入。
  - 当前选中支出，底部显示黑色短横线。
  - 右侧取消。
- 类目网格。
  - 每行 4 个类目。
  - 每个类目为浅灰圆形图标 + 类目名称。

本期支出类目：

- 餐饮。
- 购物。
- 交通。

视觉与行为：

- 为保持截图布局，可显示其他类目样式作为不可用占位，但本期只有餐饮、购物、交通可选。
- 点击餐饮、购物、交通后，类目圆形背景变为品牌黄色，并展示金额录入区。
- 点击其他支出类目时，展示“暂未实现”。
- 点击收入 Tab，切换至记账-收入类目页。
- 点击取消，返回首页。

### 3.4 记账-收入类目

页面结构与支出页一致，当前选中收入 Tab。

本期收入类目全部实现：

- 工资。
- 兼职。
- 理财。
- 礼金。
- 其他。
- 设置。

交互规则：

- 点击任一收入类目后，类目进入选中态并展示金额录入区。
- 点击支出 Tab，切换回支出类目页。
- 点击取消，返回首页。

### 3.5 记账-金额录入

触发入口：选择任一已实现类目。

页面变化：

- 类目区保留在上方，选中类目为黄色圆形背景。
- 底部出现金额录入区和数字键盘。
- 金额显示在键盘上方右侧，默认 `0.00`。
- 备注输入框位于金额下方，默认提示：`备注：点击填写备注`。
- 数字键盘包含：
  - 数字：0-9。
  - 小数点。
  - 删除键。
  - 日期按钮：默认显示 `今天`。
  - 加号、减号。
  - 黄色完成按钮。

金额规则：

- 默认金额为 `0.00`。
- 点击数字更新金额。
- 支持一位小数点。
- 点击删除键删除末尾数字。
- 金额为空或为 0 时点击完成，应提示请输入金额。
- 支出保存时金额计为负数，收入保存时金额计为正数。

备注规则：

- 点击备注输入框后进入文本输入。
- 备注可为空。
- 备注为空时，账单列表展示类目名称；备注不为空时，优先展示备注。

完成保存：

- 点击完成校验金额。
- 保存字段包括：类型、类目、金额、日期、备注、创建时间。
- 保存成功后返回首页，并刷新当前月份汇总和明细。

### 3.6 记账-选择日期弹窗

触发入口：记账金额键盘中的日期按钮。

弹窗规则：

- 当前记账页面背景加深遮罩。
- 底部弹出白色选择面板。
- 顶部操作栏：
  - 左侧：取消。
  - 中间：选择日期。
  - 右侧：确定。
- 选择器包含三列：
  - 年。
  - 月。
  - 日。
- 年、月、日均可纵向滚动。
- 默认值：`2026 / 6 / 16`。

操作规则：

- 点击取消：关闭弹窗，不改变当前账单日期。
- 点击确定：关闭弹窗，并更新键盘日期按钮。
- 当选择日期为当天时，键盘日期按钮显示 `今天`；否则显示月日格式，例如 `06月16日`。

## 5. 核心流程

### 5.1 新增支出

1. 用户在首页点击中央记账按钮。
2. 系统进入记账页，默认选中支出 Tab。
3. 用户选择餐饮、购物或交通。
4. 系统展示金额录入区。
5. 用户输入金额，可选填写备注和修改日期。
6. 用户点击完成。
7. 系统保存支出账单并返回首页。
8. 首页刷新支出合计和明细列表。

### 5.2 新增收入

1. 用户在记账页点击收入 Tab。
2. 用户选择工资、兼职、理财、礼金、其他或设置。
3. 系统展示金额录入区。
4. 用户输入金额，可选填写备注和修改日期。
5. 用户点击完成。
6. 系统保存收入账单并返回首页。
7. 首页刷新收入合计和明细列表。

### 5.3 切换首页月份

1. 用户点击首页年月区域。
2. 系统弹出月份选择器。
3. 用户滚动选择年、月。
4. 用户点击确定。
5. 系统关闭弹窗并刷新首页汇总与明细。

## 6. 验收标准

- 首页能够展示截图中的基础布局：黄色头部、月份、收入、支出、快捷入口、登录提示、日期分组流水和底部 Tab。
- 首页快捷入口和非明细 Tab 均保留入口，点击后不进入真实业务流程。
- 支出页只允许餐饮、购物、交通进入可保存流程；其他支出类目不得作为本期可用功能。
- 收入页 6 个类目均可进入金额录入流程。
- 选择餐饮后，页面展示金额、备注、数字键盘、日期按钮和完成按钮。
- 首页月份选择器支持年、月滚动选择，并有取消/确定。
- 记账日期选择器支持年、月、日滚动选择，并有取消/确定。
- 金额为 0 或空时不能保存。
- 保存成功后首页汇总和明细列表刷新。
- 所有页面的未实现入口均有明确占位反馈。
