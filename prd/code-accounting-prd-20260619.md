# CodeAccounting PRD

## 背景与目标

CodeAccounting 是一款移动端个人记账应用，对外展示名为 `Code记账`。首期目标是参考鲨鱼记账的高频记账体验，提供一个简洁、快速、可落地的本地记账版本。

首期重点是“查看当月明细”和“快速记一笔”。用户可以在首页查看指定年月的收入、支出和记账明细，可以新增支出或收入账单，并可以通过滚轮式选择器切换首页年月或账单日期。

参考截图位于 `prd/image/`，仅用于理解布局、信息层级、底部 Bar、图标风格、金额键盘和选择器弹窗。截图中的账号、登录、营销和非首期业务内容不进入本期范围。

## 范围

### 本期实现

- 首页-明细
  - 展示品牌标题 `Code记账`。
  - 展示当前选中年月。
  - 展示当月收入合计和支出合计。
  - 展示“我的记账”明细列表。
  - 支持点击年月区域弹出年月选择器。
  - 切换年月后，收入合计、支出合计和“我的记账”列表必须同步刷新。
- 底部 Bar
  - 尽量还原参考截图中的五入口结构：明细、图表、记账、发现、我的。
  - `明细` 和中央 `记账` 为本期可实现入口。
  - `图表`、`发现`、`我的` 保留入口视觉，但点击后仅展示未实现占位反馈。
  - 中央记账入口使用黄色圆形按钮和加号图标。
- 记账-支出
  - 支持支出/收入 Tab 切换。
  - 支出类目仅支持：餐饮、购物、娱乐、日用。
  - 点击任一支出类目后，该类目进入黄色选中态，并弹出金额键盘。
- 记账-收入
  - 收入类目仅支持：工资、理财、礼金、其他。
  - 点击任一收入类目后，该类目进入黄色选中态，并弹出金额键盘。
- 金额录入
  - 支持输入金额。
  - 支持填写备注。
  - 支持选择账单日期。
  - 支持完成保存。
- 日期选择
  - 首页年月选择器支持上下滑动选择年、月。
  - 记账页日期选择器支持上下滑动选择年、月、日。

### 本期仅保留入口

以下入口仅保留底部 Bar 视觉入口，不展开真实功能：

- 图表。
- 发现。
- 我的。

点击上述入口时，展示“暂未实现”提示或进入空占位页。

## 非目标

- 不实现账号、登录、注册、登出、云同步或跨设备同步。
- 不展示登录提示条。
- 首页不展示账单、预算、资产管家、购物返现、更多等快捷入口。
- 不实现预算、资产、返现、报表、发现内容、我的设置等业务流程。
- 不要求百分百复刻鲨鱼记账视觉素材；图标以高还原线性风格重绘，不直接复制受保护素材。

## 页面与交互

### 首页-明细

页面结构从上到下：

- 状态栏。
- 黄色品牌头部。
  - 顶部展示 `Code记账`。
  - 展示年月选择区，例如 `2026年`、`06月`、下拉箭头。
  - 展示收入合计，例如 `0.00`。
  - 展示支出合计，例如 `23.00`。
- 明细区域。
  - 区块标题：`我的记账`。
  - 日期分组标题，例如 `06月16日 星期二`。
  - 分组右侧展示当日支出或收入合计。
  - 账单项展示类目线性图标、名称或备注、金额。
- 底部 Bar。
  - 明细：当前选中。
  - 图表：未实现占位。
  - 记账：中央黄色圆形按钮，点击进入记账页。
  - 发现：未实现占位。
  - 我的：未实现占位。

交互规则：

- 点击年月区域，弹出首页年月选择器。
- 点击记账按钮，进入记账页，默认展示支出 Tab。
- 点击图表、发现、我的，展示“暂未实现”。
- 首页刷新时按当前选择年月汇总收入、支出和明细。
- 选择没有账单的年月时，展示空态文案：`暂无记账`。

### 首页-年月选择弹窗

触发入口：首页年月区域。

弹窗规则：

- 页面背景加深遮罩。
- 底部弹出白色选择面板。
- 顶部操作栏：
  - 左侧：取消。
  - 中间：选择月份。
  - 右侧：确定。
- 选择器包含两列：
  - 年。
  - 月。
- 年、月均可纵向滚动。
- 默认值为当前首页选中年月。

操作规则：

- 点击取消：关闭弹窗，不改变当前年月。
- 点击确定：关闭弹窗，并刷新首页收入、支出和明细。
- 滚动选择时，高亮中间选中行。

### 记账-支出类目

页面结构：

- 状态栏。
- 黄色顶部栏。
  - 中间 Tab：支出、收入。
  - 当前选中支出，底部显示黑色短横线。
  - 右侧取消。
- 类目网格。
  - 每行 4 个类目。
  - 每个类目为浅灰圆形底 + 线性图标 + 类目名称。

本期支出类目：

- 餐饮。
- 购物。
- 娱乐。
- 日用。

交互规则：

- 点击任一支出类目后，类目圆形背景变为品牌黄色，并展示金额录入区和数字键盘。
- 点击收入 Tab，切换至记账-收入类目页。
- 点击取消，返回首页。

### 记账-收入类目

页面结构与支出页一致，当前选中收入 Tab。

本期收入类目：

- 工资。
- 理财。
- 礼金。
- 其他。

交互规则：

- 点击任一收入类目后，类目圆形背景变为品牌黄色，并展示金额录入区和数字键盘。
- 点击支出 Tab，切换回支出类目页。
- 点击取消，返回首页。

### 记账-金额录入

触发入口：选择任一支出或收入类目。

页面变化：

- 类目区保留在上方，选中类目为黄色圆形背景。
- 底部出现金额录入区和数字键盘。
- 金额显示在键盘上方右侧，默认 `0.00`。
- 备注输入框位于金额下方，默认提示：`备注：点击填写备注`。
- 数字键盘包含：
  - 数字：0-9。
  - 小数点。
  - 删除键。
  - 日期按钮：默认显示 `今天`。
  - 加号。
  - 减号。
  - 黄色完成按钮。

金额规则：

- 默认金额为 `0.00`。
- 点击数字更新金额。
- 支持一位小数点。
- 点击删除键删除末尾数字。
- 金额为空或为 0 时点击完成，应提示请输入金额。
- 支出保存时金额计为负数。
- 收入保存时金额计为正数。

备注规则：

- 点击备注输入框后进入文本输入。
- 备注可为空。
- 备注为空时，账单列表展示类目名称。
- 备注不为空时，账单列表优先展示备注。

完成保存：

- 点击完成校验金额。
- 保存字段包括：类型、类目、金额、日期、备注、创建时间。
- 保存成功后返回首页，并刷新当前年月汇总和明细。

### 记账-日期选择弹窗

触发入口：金额键盘中的日期按钮。

弹窗规则：

- 当前记账页面背景加深遮罩。
- 底部弹出白色选择面板。
- 顶部操作栏：
  - 左侧：取消。
  - 中间：选择日期。
  - 右侧：确定。
- 选择器包含三列：
  - 年。
  - 月。
  - 日。
- 年、月、日均可纵向滚动。
- 默认值为当前账单日期。

操作规则：

- 点击取消：关闭弹窗，不改变当前账单日期。
- 点击确定：关闭弹窗，并更新键盘日期按钮。
- 当选择日期为当天时，键盘日期按钮显示 `今天`；否则显示月日格式，例如 `06月16日`。

## 数据与规则

### Bill

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | string | 账单唯一 ID。 |
| type | enum | `expense` 或 `income`。 |
| category | string | 类目名称。 |
| amount | decimal | 金额；支出为负，收入为正。 |
| date | date | 账单日期。 |
| remark | string | 备注，可为空。 |
| createdAt | datetime | 创建时间。 |

### 汇总规则

- 首页按选中年月筛选账单。
- 收入合计只统计正金额。
- 支出合计展示支出绝对值合计。
- 明细按账单日期倒序分组。

## 验收标准

- 首页不出现账单、预算、资产管家、购物返现、更多快捷入口。
- 首页不出现登录提示、账号提示或云同步提示。
- 首页展示 `Code记账`、年月、收入、支出和“我的记账”。
- 切换年月后，收入、支出和明细列表基于选中年月刷新。
- 底部 Bar 视觉结构尽量接近参考截图，中央记账按钮为黄色圆形加号。
- 点击底部“记账”进入支出记账页。
- 点击图表、发现、我的时不进入真实功能，仅展示未实现占位。
- 支出页只展示餐饮、购物、娱乐、日用四个类目。
- 收入页只展示工资、理财、礼金、其他四个类目。
- 点击任一类目后展示金额键盘，且选中类目为黄色选中态。
- 首页年月选择器支持年/月上下滑动选择。
- 记账日期选择器支持年/月/日上下滑动选择。
- 金额为 0 或空时点击完成，需要提示请输入金额。
- 保存成功后回到首页并刷新当前年月汇总与明细。
