# CodeAccounting PRD — 墨与纸

## 背景与目标

CodeAccounting 是一款移动端个人记账应用，对外展示名为 `Code记账`。CodeAccounting 首期记账体验，功能范围涵盖首页明细、底部 Bar、支出/收入类目、金额键盘、年月和日期选择器。

本 PRD 采用"墨与纸"极简克制风格——暖纸白底色、墨黑排版层级、鼠尾草绿品牌点缀，追求高品质手账本般的视觉体验。

设计目标：

- 建立独立品牌辨识度。
- 用排版节奏和留白替代色块填充，让信息层级更清晰。

## 范围

### 本期实现

功能范围如下：

- 首页-明细：展示品牌标题、选中年月、收入合计、支出合计和明细列表。
- 底部 Bar：五入口结构，明细和记账可用，其余为占位。
- 记账-支出：餐饮、购物、娱乐、日用四个类目。
- 记账-收入：工资、理财、礼金、其他四个类目。
- 金额录入：金额、备注、日期、删除、加减号和完成保存。
- 日期选择：首页年月选择器和记账日期选择器。

### 本期仅保留入口

- 图表、发现、我的：保留底部 Bar 视觉入口，点击时显示 `即将推出` Toast。

## 非目标

本期不实现：

- 不实现账号、登录、注册、登出、云同步或跨设备同步。
- 不展示登录提示条。
- 首页不展示账单、预算、资产管家、购物返现、更多快捷入口。
- 不实现预算、资产、返现、报表、发现内容、我的设置等业务流程。
- 图标以高还原线性风格重绘，不直接复制受保护素材。

## 品牌体系与视觉设计系统

### 设计风格

墨与纸——像一本高品质的手账本。纸白底色、墨黑字体、鼠尾草绿点睛。所有信息层级通过字号、字重和留白表达，不依赖大面积色块填充。

### 色彩系统

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| 纸白 Background | `#FAFAF8` | 页面底色、卡片内部 |
| 墨黑 Primary Text | `#1A1A1A` | 标题、金额、主要文字 |
| 中灰 Secondary Text | `#8C8C88` | 次级信息、日期、类目名 |
| 浅灰 Divider | `#E8E8E4` | 分隔线、卡片描边、非活跃元素 |
| 鼠尾草绿 Brand | `#7D9B76` | 选中态、CTA 按钮、收入金额 |
| 淡绿 Brand Light | `#EDF3EB` | 选中类目底色、轻量高亮 |
| 警示红 Expense Accent | `#C75C5C` | 支出金额数字 |

### 字体策略

- 金额数字：等宽字体（推荐 DIN Next 或 SF Mono），确保数字对齐和精确感。
- 正文：系统默认（Android: Roboto / HarmonyOS Sans；iOS: SF Pro）。
- 层级方式：仅靠字号（14/16/20/28/36）和字重（Regular/Medium/Bold）区分，不依赖颜色区分层级。

### 圆角与间距

- 卡片圆角：`12px`。
- 按钮圆角：`24px`（胶囊形）。
- 类目图标圆角：`16px`（圆角方形）。
- 页面水平边距：`20px`。
- 卡片间距：`12px`。
- 元素间距优先使用 `8px` 倍数。

### 图标风格

- 纯线条（stroke width 1.5px），不填充。
- 无圆形底色包裹。
- 图标容器：`24x24pt`。
- 选中时：图标下方出现淡绿 `#EDF3EB` 圆角方形底色。

## 页面与交互

### 首页-明细

页面结构从上到下：

- 状态栏：系统默认，深色图标。
- 顶部区域（纸白底直通，无背景色块）：
  - 左上：`Code记账`，16px Medium，墨黑 `#1A1A1A`。
  - 右上：无按钮。
- 月份选择行：
  - 左侧：大号月份 `6月`，36px Bold，墨黑。
  - 月份右侧：向下小箭头 `▾`，中灰 `#8C8C88`，点击触发年月选择器。
  - 同行右侧：年份 `2026`，14px Regular，中灰。
- 收支摘要卡片：
  - 纸白底 + `#E8E8E4` 细描边（1px）圆角卡片（圆角 `12px`）。
  - 水平二等分，竖向细线 `#E8E8E4` 分隔。
  - 左半：`支出` 标签（14px 中灰）+ 金额（24px Bold 警示红 `#C75C5C`），如 `¥23.00`。
  - 右半：`收入` 标签（14px 中灰）+ 金额（24px Bold 鼠尾草绿 `#7D9B76`），如 `¥0.00`。
- 明细区域：
  - 标题：`明细`，16px Medium，墨黑，左对齐。
  - 日期分组头：`6月16日 周二`，14px Regular，中灰；右侧当日支出合计，14px，中灰。
  - 账单条目：左侧类目线条图标（24x24，无圆形底）+ 12px 间隔 + 类目名或备注（16px Regular 墨黑）+ 右侧金额（16px Medium，支出警示红 `-¥23`，收入鼠尾草绿 `+¥50`）。
  - 条目之间：无分隔线，靠 12px 垂直间距区分。
  - 空态：居中 `暂无记账`，14px，中灰，无插画。
- 底部 Bar。

交互规则：

- 点击月份区域（含箭头），弹出首页年月选择器。
- 点击记账按钮，进入记账页，默认展示支出 Tab。
- 点击图表、发现、我的，显示 `即将推出` Toast，1.5 秒自动消失。
- 首页刷新时按当前选择年月汇总收入、支出和明细。
- 选择没有账单的年月时，展示空态。

### 底部 Bar

结构：

- 背景：纸白 `#FAFAF8`，顶部极细线 `#E8E8E4`（0.5px）。
- 5 个入口等宽排列：明细、图表、记账、发现、我的。

样式：

- 未选中：图标和文字均为中灰 `#8C8C88`，文字 12px。
- 选中（明细）：图标和文字均为墨黑 `#1A1A1A`，文字 12px Medium。
- 记账按钮：不随选中态变化；固定为 `44x44` 鼠尾草绿 `#7D9B76` 圆角方形（圆角 `12px`），内部白色加号图标；无上浮、无阴影、不突破 Bar 边界。
- 占位入口：视觉与未选中一致，点击时显示 `即将推出` Toast。

### 首页-年月选择弹窗

触发入口：首页月份区域。

弹窗规则：

- 遮罩：`#1A1A1A` 40% 透明度。
- 底部弹出纸白面板，顶部圆角 `16px`。
- 操作栏：
  - 左：`取消`，16px Regular，中灰 `#8C8C88`。
  - 中：`选择月份`，16px Medium，墨黑。
  - 右：`确定`，16px Medium，鼠尾草绿 `#7D9B76`。
  - 操作栏底部极细线 `#E8E8E4`。
- 选择器两列：年、月。
- 选中行：20px Bold 墨黑，行上下各一条极细线 `#E8E8E4` 框定。
- 非选中行：16px Regular 中灰，随距离渐隐。
- 默认值为当前首页选中年月。

操作规则：

- 点击取消：关闭弹窗，不改变年月。
- 点击确定：关闭弹窗，刷新首页收入、支出和明细。

### 记账-支出类目

页面结构：

- 状态栏。
- 顶部栏：纸白底，底部极细线。
  - 中间 Tab：`支出` / `收入`。
  - 选中 Tab：墨黑 `#1A1A1A`，16px Medium，下方 2px 鼠尾草绿短横线（宽度 = 文字宽度）。
  - 未选中 Tab：中灰 `#8C8C88`，无下划线。
  - 右侧：`取消`，16px Regular，中灰。
- 类目网格：
  - 每行 4 个类目，水平等距，页面水平边距 `20px`。
  - 每个类目：48x48 线条图标（默认中灰 `#8C8C88`）+ 类目名称（13px Regular 中灰）。
  - 无圆形或方形底色包裹。
  - 类目间垂直间距 `24px`。

本期支出类目：餐饮、购物、娱乐、日用。

交互规则：

- 点击任一支出类目后，图标下方出现 `48x48` 淡绿 `#EDF3EB` 圆角方形底色（圆角 `12px`），图标变鼠尾草绿，名称变墨黑，底部弹出金额录入区和键盘。
- 点击收入 Tab，切换至收入类目页。
- 点击取消，返回首页。

### 记账-收入类目

页面结构与支出页一致，当前选中收入 Tab。

本期收入类目：工资、理财、礼金、其他。

交互规则与支出页对称。

### 记账-金额录入

触发入口：选择任一支出或收入类目。

页面变化：

- 类目区保留在上方，选中类目为淡绿底 + 绿色图标。
- 底部推出金额录入区和数字键盘。

金额录入区：

- 纸白底，与键盘区之间极细线分隔。
- 右侧金额显示：36px Bold 等宽字体。
  - 默认 `¥0.00`，中灰 `#8C8C88`。
  - 输入中：墨黑 `#1A1A1A`，鼠尾草绿竖线光标闪烁。
- 左侧备注输入：`添加备注`，14px Regular，中灰提示。

数字键盘：

- 纸白底，无格线、无按键边框。
- 4 列 x 4 行布局：

```
 7      8      9      📅 今天
 4      5      6        +
 1      2      3        −
 .      0      ⌫      完成
```

- 数字键：20px Medium 墨黑，点击时背景闪现 `#E8E8E4` 淡灰圆角方形反馈（150ms）。
- 删除键：线条图标，中灰。
- 日期按钮：小日历线条图标 + `今天` 文字（14px Regular 中灰）；非当天显示月日格式。
- 加号/减号：20px 中灰。
- 完成按钮：鼠尾草绿 `#7D9B76` 填充圆角方形（圆角 `12px`），白色 `完成` 文字（16px Medium）。

金额规则：

- 默认 `0.00`，点击数字更新。
- 支持一位小数点。
- 删除键删除末尾数字。
- 金额为 0 点击完成时 Toast 提示 `请输入金额`。
- 支出保存时金额计为负数，收入为正数。

备注规则：

- 点击后进入文本输入，可为空。
- 为空时列表展示类目名称，不为空时优先展示备注。

完成保存：

- 校验金额后保存，字段：类型、类目、金额、日期、备注、创建时间。
- 成功后返回首页并刷新当前年月汇总与明细。

### 记账-日期选择弹窗

触发入口：金额键盘日期按钮。

弹窗规则：

- 遮罩：`#1A1A1A` 40% 透明度。
- 底部弹出纸白面板，顶部圆角 `16px`。
- 操作栏：
  - 左：`取消`，16px Regular，中灰。
  - 中：`选择日期`，16px Medium，墨黑。
  - 右：`确定`，16px Medium，鼠尾草绿。
  - 操作栏底部极细线。
- 选择器三列：年、月、日。
- 选中行/非选中行样式与年月选择器一致。
- 默认值为当前账单日期。

操作规则：

- 点击取消：关闭弹窗，不改变账单日期。
- 点击确定：关闭弹窗，更新键盘日期按钮。当天显示 `今天`，否则显示月日格式。

## 数据与规则

### Bill

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | string | 账单唯一 ID。 |
| type | enum | `expense` 或 `income`。 |
| category | string | 类目名称。 |
| amount | decimal | 金额；支出为负，收入为正。 |
| date | date | 账单日期。 |
| remark | string | 备注，可为空。 |
| createdAt | datetime | 创建时间。 |

### 汇总规则

- 首页按选中年月筛选账单。
- 收入合计只统计正金额。
- 支出合计展示支出绝对值合计。
- 明细按账单日期倒序分组。

## 验收标准

- 首页不出现账单、预算、资产管家、购物返现、更多快捷入口。
- 首页不出现登录提示、账号提示或云同步提示。
- 首页展示 `Code记账`（左上 16px），大号月份、年份、收支摘要卡片和明细列表。
- 收支摘要卡片中支出金额为警示红 `#C75C5C`，收入金额为鼠尾草绿 `#7D9B76`。
- 切换年月后，收入、支出和明细列表基于选中年月刷新。
- 底部 Bar 记账按钮为 `44x44` 鼠尾草绿圆角方形（内嵌，不突出 Bar）。
- 点击图表、发现、我的时显示 `即将推出` Toast。
- 支出页只展示餐饮、购物、娱乐、日用四个类目。
- 收入页只展示工资、理财、礼金、其他四个类目。
- 类目无圆形底色包裹，选中时出现淡绿圆角方形底。
- 数字键盘无格线无边框，完成按钮为鼠尾草绿。
- 首页年月选择器支持年/月上下滑动选择。
- 记账日期选择器支持年/月/日上下滑动选择。
- 弹窗确定按钮为鼠尾草绿文字。
- 金额为 0 或空时点击完成，提示 `请输入金额`。
- 保存成功后回到首页并刷新当前年月汇总与明细。
